Detailed Description

How it works:

The MAC adderess is the key to the table, it is used as the index. All the events (last ones + one deep stack) are stored under the key, and we consult this table for telling what is on the air.

RF packets come in that have been seen are discarded. If an RF packet has to be pushed through, (ex: statuses) it receives a random number in one of the fields, so it is unique and it goes through. see: qcan_main.c line 202
